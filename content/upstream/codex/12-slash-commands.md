# 12 · 斜杠命令与快捷键：会话里的「快捷操作面板」

> 📚 **系列导航**：上一篇〔[11 项目说明书 AGENTS.md](11-agents-md.md) 〕教你把规矩写进文件、让 Codex 每轮开工都先读一遍。那是「事先立规矩」；这一篇讲「**当场拨开关**」——进了会话之后，一个 `/` 能让你切模型、清上下文、看状态、审改动，再配上一把顺手的快捷键，操作效率直接翻倍。下一篇〔[13 提示词（Prompt）写法](13-prompting.md) 〕再回到「怎么把需求说清楚」。

我数了一遍 Codex 官方文档里列的斜杠命令。

CLI 那张表，整整 **40 多条**；桌面 App 列了 **6 条**；IDE 扩展列了 **8 条**。我第一次翻 CLI 那张表的时候是真有点懵——光一个终端会话里，能用斜杠当场调的开关就这么多，而我之前来回用的，**满打满算不超过五个**。

更扎心的是后面这件事。我有阵子每次想清空对话重开，都老老实实 `Ctrl+C` 退出、再敲 `codex` 重启，等它重新加载一遍项目——一套下来小十秒。这么连着干了快两周，直到某天翻文档撞见 `/clear`，一句话写着「清掉终端、开一段全新对话」。**原来在输入框打六个字符就行，那两周的重启全是白费功夫。**

说这段是想让你别走我的弯路：**斜杠命令不是「高级玩法」，它是 Codex 会话里最基础的控制台**。你想对 Codex 本身做点啥——不是让它写代码，而是调教它当前这次会话——入口几乎都在这一个 `/` 后面。这一篇把 CLI、桌面 App、IDE 三处的斜杠命令和常用快捷键一次摊开，给你一张能直接抄的速查。

**看完这一篇，你会拿到：**

- 一句话说清斜杠命令是啥、为什么三个入口的命令数量差这么多
- 一张「按你在干哪一步」分组的 CLI 常用命令速查（`/model`、`/status`、`/diff`、`/compact`、`/review`、`/init`……），不用背、按需查
- 桌面 App 和 IDE 扩展各自能用哪几个斜杠命令——别拿 CLI 那张大表去套，会扑空
- 一份最该记的 CLI 键盘快捷键表（`Ctrl+L`、`Ctrl+O`、`Tab`、`Ctrl+G`、`!` 前缀……），全对照官方核对过
- 自定义快捷键怎么改（`/keymap`），以及为什么新手先别急着改
- 一个能照着跑、给了预期输出的最小验证流程

> ⚠️ 下文凡涉及具体命令、快捷键、默认行为，都以 Codex [官方文档](https://developers.openai.com/codex) 为准；模型名、版本号、退出键这类随版本或终端变的东西，**以你本地界面实际显示为准**，本篇不写死。桌面 App 的快捷键官方目前**只标了 macOS**，Windows 键位看本机设置面板，下面会专门提醒。

## 01 先搞懂：斜杠命令到底是个啥，凭什么只在开头算数

先给结论：**斜杠命令就是你在 Codex 会话里打的「控制指令」——不是说给模型听让它干活，而是直接命令 Codex 这个程序：切模型、清上下文、看状态、跑个流程。**

你回想前面几篇，跟 Codex 打交道一直分两种话。一种是**正经需求**：「帮我把这个函数重构一下」「这段报错咋回事」——这是说给模型听的。另一种是**元操作**：「把对话清了重开」「换个更省的模型」「让我看看现在啥配置」——这些不该靠跟模型「聊」，而该有个**直接的开关**。斜杠命令就是这批开关的统一入口。官方对它的定位讲得很干脆——「斜杠命令让你对 Codex 有快速、键盘优先的控制」。

**类比：写字楼前台那块「分机号一览表」。** 你进一栋写字楼办事，不会逮着前台从头解释「我要找三楼那个管报销的」——你扫一眼墙上贴的分机表，财务 8101、人事 8102、IT 8105，直接拨。**每个号通一个确定的部门，拨过去立刻接通，不用绕、不会理解歪。** 斜杠命令就是 Codex 会话的这张分机表：`/model` 通「换模型」、`/status` 通「看状态」、`/diff` 通「审改动」——打下去执行的是程序写死的固定动作，跟「求模型帮忙」是两码事。

这里有个**新手最容易栽的点**：斜杠命令得让 `/` 当这条消息的**第一个字符**才算数。你在输入框打 `/`，菜单立刻弹出来；接着多打几个字母，它实时筛选。官方在三个入口的文档里都是同一句操作——「在输入框打 `/`，从列表里选，或继续打字来过滤（比如 `/status` ）」。

落到你真实会遇到的几个瞬间，体会一下这张「分机表」啥时候拨：

- **聊着聊着发现模型不够强**——不用退出重开，CLI 里打 `/model` 当场换上更强的，对话不断。
- **不确定现在是什么配置**——打 `/status`，当前模型、审批策略、上下文余量一屏看清。
- **一个任务干完想换个全新的活儿**——CLI 里打 `/clear` 把台面擦干净重开。

这几件事的共同点：**都不是「让 Codex 写代码」，而是「调教这次会话本身」**——这正是斜杠命令的主场。

那为什么三个入口命令数量差这么大（CLI 四十多、App 六个、IDE 八个）？一句话：**CLI 是 Codex 功能最全的那张脸**（第 08 篇说过），很多偏底层、偏调试的开关只在终端给；桌面 App 和 IDE 是图形界面，该用按钮、菜单、设置面板干的事就不挤进斜杠列表了。所以**别拿 CLI 那张大表去 App 或 IDE 里套**，各认各的，下面分开讲。

> 💡 一句话总结：斜杠命令是 Codex 会话的「分机号一览表」——管的是程序本身（切模型、清上下文、看状态），不是给模型派活；**它只在消息开头才算数，打一个 `/` 就能看到你当前入口真正能用的全部命令**。

## 02 CLI 常用命令：不用背，按「你在干哪一步」来查

CLI 那四十多条，**真要一个个背，纯属跟自己过不去**。好在它们能按「一次会话的典型流程」归类——你处在哪一步，自然就该用哪几个。我把日常高频的按这个思路理成四组，**对着「你现在想干啥」找那一行就行**，全部对照官方 `cli/slash-commands` 核对过。

先说清一件事：第 08 篇已经把 CLI 的交互界面、最该记的几个救急命令点过了，这一节是**把那张表铺全、分组**，不是重复——第 08 篇点到为止的，这里给你能查的全集思路。

### 第一组：搭场子、定规矩

| 命令 | 干啥的 | 你啥时候用 |
|------|--------|-----------|
| `/init` | 在当前目录生成一份 `AGENTS.md` 脚手架 | 第一次在某个仓库里立规矩（详见第 11 篇） |
| `/permissions` | 设 Codex 不用问就能干哪些事 | 想在 Auto / Read Only 之间收紧或放开 |
| `/mcp` | 列出已配置的 MCP 工具（加 `verbose` 看详情） | 接外部服务、排查工具连没连上 |
| `/skills` | 浏览并选用一个本地 Skill | 想让某个专项 Skill 接管下一轮 |
| `/status` | 看会话配置和 token 用量 | 确认当前模型、审批策略、可写目录、上下文余量 |

进新项目基本是固定套路：先 `/init` 让它读一遍代码、吐一份 `AGENTS.md` 草稿（第 11 篇专讲怎么写好它），再 `/permissions` 把动手权限调到你舒服的档。

### 第二组：干活干到一半，调状态

| 命令 | 干啥的 | 你啥时候用 |
|------|--------|-----------|
| `/model` | 切换模型（有的还能调推理强度） | 想换更强或更快的模型 |
| `/compact` | 把可见对话压缩成摘要、腾出 token | 聊太久、上下文快满了 |
| `/clear` | 清掉终端 + 开一段全新对话 | 换个全新任务，台面收拾干净 |
| `/new` | 在同一个 CLI 会话里开一段新对话 | 换任务，但**不想先清屏** |
| `/plan` | 切到 plan 模式，可顺带带一句提示 | 大改动前先让它出方案不动手 |

这里有三个**最容易混**的，专门掰一下，差别全来自官方：

**`/clear` 和 `/new` 都开新对话，区别在「要不要先擦屏幕」。** 官方原话：`/new` 不像 `/clear` 那样先清掉当前终端画面。所以**想连旧记录一起眼不见心不烦用 `/clear`，想留着上面的滚屏只换上下文用 `/new`**。

**`/clear` 和 `/compact` 又是另一对**——一个清空、一个压缩。换个不相干的新活儿用 `/clear` （台面全擦），**同一个任务但聊太长了用 `/compact`** （把草稿纸整理成一页要点接着干）。我前面栽的那两周「重启大法」，本质就是没分清「重启」和「清空对话」——其实 `/clear` 一步就到。

### 第三组：活儿交出去之前，检查

| 命令 | 干啥的 |
|------|--------|
| `/diff` | 看 Git diff，**含 Git 还没开始跟踪的新文件** |
| `/review` | 让 Codex 审一遍你的工作区改动（盯行为变化和漏掉的测试） |
| `/copy` | 复制 Codex 最近一条**已完成**的输出（也可按 `Ctrl+O`） |

`/diff` 我几乎每次提交前都按——它连**还没 `git add`、Git 压根没跟踪的新文件**都列出来，比我自己 `git status` 加 `git diff` 两条命令一把抓得还全。官方明说了它会同时显示已暂存、未暂存、和未跟踪三类改动。

### 第四组：杂项里你早晚用得上的

| 命令 | 干啥的 |
|------|--------|
| `/keymap` | 查看并修改 TUI 快捷键绑定，改动可持久化（详见第 05 节） |
| `/statusline` | 交互式挑选、排序状态栏（页脚）显示哪些字段 |
| `/theme` | 选一个语法高亮主题并存下来 |
| `/resume` | 从会话列表恢复一段存过的对话 |
| `/side`（别名 `/btw`） | 开一段「侧聊」临时分支，不污染主线 |
| `/fork` | 把当前对话分叉成一个新线程，原对话不动 |
| `/quit`（别名 `/exit`） | 退出 CLI（**退之前先保存或提交重要改动**） |

这四组**没给全**（官方那张表四十多行，要查全的回头翻 `cli/slash-commands` ），但**覆盖了你日常九成的场景**。剩下的临时想干啥，在输入框打个 `/` 加几个字母筛一下，比背一张表实在得多。

还有个 Codex 挺贴心的细节，官方专门写了：**任务正跑着的时候，你可以先把一条斜杠命令打好、按 `Tab` 排进队列**，等这一轮跑完它自动解析、接着处理——命令菜单和报错会在当前回合结束后才冒出来。我经常在它跑测试的间隙，先把 `/review` 用 `Tab` 排好队，一跑完审查无缝衔接，省了来回干等。

> 💡 一句话总结：CLI 命令别硬背，**按「你在哪一步」分组查**——搭场子（`/init` `/permissions` `/mcp`）、干活中（`/model` `/compact` `/clear`）、交付前（`/diff` `/review`）、杂项（`/keymap` `/resume` `/fork`）；记牢 `/clear`（清屏+换上下文）、`/new`（不清屏换上下文）、`/compact`（压缩续聊）这三兄弟的差别。

## 03 桌面 App 与 IDE 扩展：命令少得多，别拿 CLI 那张表去套

第 08 篇我提醒过一句——**别拿 CLI 四十多条的认知去桌面 App 和 IDE 里套，会扑空**。这一节就把这俩各自的斜杠命令单独列清楚，省得你在 App 里打 `/keymap` 半天没反应还以为坏了。

为啥它俩命令少这么多？前面说了：图形界面里很多事是用按钮、菜单、设置面板干的，没必要全塞进斜杠列表。**类比：同一家银行的手机 App 和柜台。** 柜台（CLI）什么业务都能办、连最冷门的也行；手机 App（桌面 App / IDE）只把高频那几样做成首页大按钮，剩下的引导你去别处。**不是 App 残废，是常用动作前置、长尾动作搬走了。**

### 桌面 App：官方目前列这 6 条

| 斜杠命令 | 作用 |
|---------|------|
| `/status` | 看线程 ID、上下文用量、限额情况 |
| `/review` | 开代码审查模式，审未提交改动或跟基线分支比 |
| `/plan` | 切到 plan 模式，做多步规划 |
| `/goal` | 设一个 Codex 要持续达成的目标（先用 `/plan` 把它捋清楚再设） |
| `/mcp` | 看已连接的 MCP server 状态 |
| `/feedback` | 提反馈（可附带日志） |

> 两个补充，都来自官方：① 在输入框打 `$` 能**显式调用 Skills**，启用的 Skills 也会出现在斜杠列表里；② `/goal` 要是列表里没有，可能得在 `config.toml` 里开 `features.goals = true`，或跑 `codex features enable goals` 来开启。

桌面 App 真正的「快捷」其实在**键盘快捷键**上——清空对话、新建线程、切面板这些，都走 `Cmd` 系快捷键而不是斜杠命令（第 07 篇那张快捷键表给过，这里不重复）。所以在 App 里，**斜杠管「下指令式的动作」（审查、规划、看状态），快捷键管「界面操作」（开窗、切栏、搜历史）**，分工和 CLI 不太一样。

### IDE 扩展（VS Code 等）：官方列这 8 条

| 斜杠命令 | 作用 |
|---------|------|
| `/status` | 看线程 ID、上下文用量、限额情况 |
| `/review` | 开代码审查模式，审未提交改动或跟基线分支比 |
| `/goal` | 设一个 Codex 要持续达成的持久目标 |
| `/auto-context` | 开 / 关「自动上下文」，自动带上最近文件和 IDE 上下文 |
| `/local` | 切到本地模式，在你的工作区里跑任务 |
| `/cloud` | 切到云端模式，把任务跑到远端（需云端权限） |
| `/cloud-environment` | 选用哪个云端环境（仅云端模式下可用） |
| `/feedback` | 提反馈（可附带日志） |

IDE 这套里最有「IDE 特色」的是 `/local` 和 `/cloud` 这对——**一句话就能把当前任务从「在我电脑上跑」切到「丢云端跑」**（第 09、10 篇分别讲过本地和云端两条线）。我在 VS Code 里调试一个要跑很久的重构时，常先 `/local` 验证思路对不对，确认了再 `/cloud` 扔上去挂着，本机腾出来干别的。

把三个入口的斜杠命令摆一起对比，差异一目了然：

| 入口 | 斜杠命令数量（官方现列） | 特色命令 | 其余操作靠什么 |
|------|------------------------|---------|--------------|
| **CLI** | 40 多条 | `/keymap` `/fork` `/side` `/statusline` 等一大批 | 键盘快捷键（第 05 节） |
| **桌面 App** | 6 条 | `/goal` `/plan` | `Cmd` 系快捷键 + 按钮（第 07 篇） |
| **IDE 扩展** | 8 条 | `/local` `/cloud` `/auto-context` | VS Code 命令面板 + 快捷键（第 09 篇） |

> ⚠️ 这三个数量是官方文档**当前列出的**条目，会随版本变。最靠谱的办法永远是：**在你正用的那个入口打一个 `/`，看它实际弹出来的列表**——那才是你这台机器、这个版本真能用的全集，别拿别人截图或这张表死磕。

> 💡 一句话总结：桌面 App 现列 6 条、IDE 现列 8 条，都比 CLI 那四十多条少得多——**图形界面把高频动作前置成按钮、把界面操作交给快捷键，斜杠只留「下指令式」的那几个**；IDE 独有 `/local`、`/cloud` 一键切运行位置；具体能用啥，打 `/` 当场看。

## 04 IDE 里还有一类「命令」：VS Code 命令面板里的 Codex 动作

上一节讲的是 IDE **聊天框里**打 `/` 的斜杠命令。但 IDE 扩展还藏着另一套你容易忽略的「命令」——**VS Code 命令面板（Command Palette）里的 Codex 动作**，这俩不是一回事，得分清。

先说它解决什么。有些操作发生在**编辑器层面**，不在聊天框里：比如选中一段代码想「丢给当前线程当上下文」、或者光标停在一句 `// TODO` 上想「让 Codex 把这条实现掉」。这些动作没法用斜杠命令触发（斜杠在聊天框里），得走命令面板或绑成快捷键。

**类比：右键菜单 vs 聊天窗。** 聊天框里的斜杠命令像你在对话里下指令；命令面板里的 Codex 动作像你**在代码上点右键选「交给 Codex」**——一个是「说」，一个是「指着某段代码动手」。两套入口，配合着用。

官方在 `ide/commands` 列了这几个扩展命令（命令 ID 给你了，方便绑快捷键）：

| 命令（面板里搜 `Codex` 或命令 ID） | 默认快捷键 | 干啥的 |
|----------------------------------|-----------|--------|
| `chatgpt.newChat` | macOS `Cmd+N` / Win·Linux `Ctrl+N` | 新建一个线程 |
| `chatgpt.addToThread` | 无（可自绑） | 把**选中的文本范围**加进当前线程当上下文 |
| `chatgpt.addFileToThread` | 无（可自绑） | 把**整个文件**加进当前线程当上下文 |
| `chatgpt.implementTodo` | 无（可自绑） | 让 Codex 处理选中的那条 TODO 注释 |
| `chatgpt.newCodexPanel` | 无（可自绑） | 新建一个 Codex 面板 |
| `chatgpt.openSidebar` | 无（可自绑） | 打开 Codex 侧边栏面板 |

怎么给它们绑快捷键？官方给的步骤照抄：

1. 开命令面板（macOS `Cmd+Shift+P` ，Windows / Linux `Ctrl+Shift+P` ）。
2. 跑 **Preferences: Open Keyboard Shortcuts**（偏好设置：打开键盘快捷方式）。
3. 搜 `Codex` 或具体命令 ID（比如 `chatgpt.newChat` ）。
4. 点那支铅笔图标，输入你想要的快捷键。

我自己给 `chatgpt.addToThread` 绑了个顺手的键——选中一段看不懂的代码，一按就丢进侧边栏的线程里，比复制粘贴再切窗口快多了。**这套「编辑器动作」是 IDE 扩展相对 CLI 多出来的便利**，纯终端里没有「选中代码右键交给它」这种操作。

> 💡 一句话总结：IDE 扩展有**两套**入口——聊天框里的**斜杠命令**（下指令）和命令面板里的 **Codex 动作**（`chatgpt.addToThread`、`implementTodo` 等，指着代码动手）；后者默认大多没绑键，按官方四步在「键盘快捷方式」里自己绑一个最顺手。

## 05 CLI 键盘快捷键：比斜杠更快的那一层

斜杠命令已经够快了，但有一小撮高频操作，**连打斜杠都嫌慢**——按一个键就完事。这一节专讲 CLI 里这些纯键盘动作。第 08 篇给过一张救急表，这里把它和官方 `cli/slash-commands` 里散落提到的快捷键合并、补全。

**类比：键盘快捷键之于斜杠命令，就像双击之于「右键→打开」。** 右键菜单（斜杠命令）什么都能选，但常用的那几样，你早练成了双击、`Ctrl+C` 、`Ctrl+V` 的肌肉记忆——**手指比眼睛快**。下面这些就是 Codex 终端里最该练成肌肉记忆的：

| 键盘动作 | 作用 | 对应的斜杠命令 |
|---------|------|--------------|
| `Ctrl+L` | **只清屏幕、保留当前对话**（强制重绘） | 无（和 `/clear` 不一样，见下） |
| `Ctrl+O` | 复制 Codex 最近一条**已完成**的输出 | 等同 `/copy` |
| `Tab`（任务跑着时） | 把下一条输入 / 斜杠命令 / `!` 命令排进队列 | 无 |
| `Ctrl+G` | 在外部编辑器里写长提示词，写完带回输入框（编辑器由 `VISUAL` 或 `EDITOR` 环境变量决定，没设过的话可能默认是 vi——想用 VS Code 先跑 `export VISUAL=code`） | 无 |
| `Alt+R` | 切换 raw 滚屏模式（让终端选中、复制更直接） | 等同 `/raw` |
| `!` 前缀（行首） | 不打扰 Codex，顺手在终端跑条 shell 命令 | 无 |

有三处**官方专门强调、最容易按错或用错**的，单独说：

**`Ctrl+L` 不等于 `/clear`。** 这是 Codex 里最容易混的一对，官方原话写得很直白：`Ctrl+L` 只清终端画面、**保留当前对话**；而 `/clear` 是清屏 + **开一段全新对话**（上下文一起没了）。一个是「擦黑板」，一个是「换张新黑板」。另外，任务正在跑的时候这两个都按不了——官方特意说了，两者都会在任务进行中被禁用，等跑完再按。我去年在一台 SSH 连过去的服务器上跑 Codex，网络抖了一下整个界面错位，差点 `Ctrl+C` 杀掉重来——幸好想起 `Ctrl+L`，一按就重绘干净，**那一长串上下文一点没丢**。要是真重启，前面攒的全白攒。

**`!` 前缀——顺手跑命令，还让 Codex「看见」结果。** 在输入框行首打 `!`，后面跟一条命令（比如 `!git status` ），这条直接在你终端跑。官方说 Codex 会**把它的输出当成「用户提供的命令结果」收进上下文**，而且照样走你的审批和沙箱设置。它香在哪？想瞄一眼仓库状态、又不想让 Codex 专门跑一趟烧 token——`!git status` 立等可取，下一句它还能接着这个状态聊。

```text
!git status
```

**`Ctrl+O` 复制输出，有两个边界。** 官方说：要是当前回合还在跑，`Ctrl+O` 复制的是**最近一条已完成**的输出（不是正在生成的那条）；而且在 Codex 出第一条完整输出之前、以及刚回滚之后，这个动作**不可用**。知道这俩边界，就不会纳闷「为啥按了没反应」。

### 想改键？用 `/keymap`，但新手先别急

默认键不顺手能改。CLI 提供了 `/keymap`——查看、修改、并把自定义绑定持久化进 `config.toml` 的 `tui.keymap` 。官方给的用法：打 `/keymap` → 挑要改的快捷键上下文和动作 → 输入新绑定或清掉旧的。键名长这样：`ctrl-a` 、`shift-enter` 、`page-down` 。

不过我得劝一句——**新手先把默认的用熟，别一上来就折腾配置**。我自己是用了小半年、确实有两个键和我别的工具冲突了，才去改的。一上来就重绑，回头看文档、问别人，你的键位跟所有教程都对不上，反而给自己添乱。

> 💡 一句话总结：CLI 键盘动作里 `Ctrl+L`（清屏不丢对话，**别和 `/clear` 搞混**）、`Ctrl+O`（复制输出）、`Tab`（排队）、`Ctrl+G`（开编辑器写长指令）、`!`（顺手跑命令还让 Codex 看见）最常用；想改键有 `/keymap`，但先把默认的练成肌肉记忆。

## 06 动手：把三个入口的斜杠命令各跑一遍

光看不练记不住。下面给一套**最小验证流程**，不依赖任何复杂项目，新建个空文件夹就能跑通 CLI 那段；App 和 IDE 那段在你装好的对应入口里点一下即可。

> 平台差异先说清：下面建目录的 `mkdir` 在 **Mac / Linux** 直接用；**Windows** 用 PowerShell 把 `mkdir -p` 换成 `mkdir`，或在资源管理器里手动建。还没装 Codex CLI 的回第 03 篇装好再来。

### CLI 部分（五步，全程在终端）

**第一步：建个空目录，进去启动。**

```bash
mkdir -p ~/codex-slash-demo && cd ~/codex-slash-demo && git init
codex
```

**预期**：进入全屏 TUI，最底下是输入框，贴着输入框有状态栏小字。

**第二步：打一个 `/` 看你这台机器的真实命令全集。**

在输入框只打一个斜杠（**别回车**）：

```text
/
```

**预期**：弹出一长串命令菜单，旁边带各自说明。**这张菜单就是你当前版本真能用的全集**——你会看到它比本篇任何一张表都长（官方一直在加命令）。再多打几个字母（比如 `/st` ），列表会实时筛到 `/status` 这些上。

**第三步：用 `/status` 看一眼当前配置。**

```text
/status
```

**预期**：Codex 打印当前会话概况——用的哪个模型、审批策略是啥、有哪些可写目录、上下文还剩多少。**心里先有个底。**

**第四步：试 `Ctrl+L` 和 `!`，体会「快捷键比斜杠还快」。**

先随便让屏幕滚几行（比如打个 `!ls` 回车），再按 `Ctrl+L` ：

```text
!ls
```

**预期**：`!ls` 那条直接在终端列出当前目录内容（这会儿目录是空的，列出来基本没东西很正常），且这条命令和输出被收进了上下文；按 `Ctrl+L` 后屏幕被清干净重绘，但**你刚才的对话一条没少**（往上滚还在）——这就是它和 `/clear` 的区别。

**第五步：试 `/diff`（顺便让它真干件活），再退出。**

先让它干点会改文件的小事：

```text
新建一个文件 hi.txt，里面写一行 hello from codex
```

它可能停下来请求批准（取决于当前审批策略），同意即可。完事后：

```text
/diff
```

**预期**：Codex 列出 Git 视角的改动，**包括这个还没被 Git 跟踪的新文件 `hi.txt`** ——这正是 `/diff` 比手敲 `git status` 全的地方。看完退出：

```text
/exit
```

**预期**：退回普通终端（`/quit` 同效，`Ctrl+C` 也能中断）。

### 桌面 App / IDE 部分（各一步，验证「命令表不一样」）

**桌面 App**：打开 App，在某个线程的输入框打一个 `/` 。

**预期**：弹出的列表**短得多**，大致就是 `/status`、`/review`、`/plan`、`/goal`、`/mcp`、`/feedback` 这几条（以你本地为准）。亲眼确认它和 CLI 那张大表不是一回事。再打个 `$` ，会看到它列出可调用的 Skills。

**IDE 扩展（VS Code 等）**：在 Codex 聊天输入框打 `/` 。

**预期**：列表里能看到 `/local`、`/cloud`、`/auto-context` 这些 **CLI 和 App 都没有**的命令。这就直观证明了第 03 节那句——**三个入口的斜杠命令各认各的**。

跑完这一圈，你就把本篇最核心的认知——**「打 `/` 看真实全集」「三个入口命令表不同」「`Ctrl+L` ≠ `/clear`」「`/diff` 连未跟踪文件都看得到」**——亲手验证了一遍，比记十条文档都实在。

> ⚠️ 小插曲：CLI 第五步它要是没停下来问、直接建好了，说明你当前审批策略比较宽松（Auto 档下在工作区内写文件本就不拦）——这不是 bug，是第 02 篇讲的沙箱 + 审批在按规则放行。想看它「停下来问」那一下，用 `/permissions` 切到 `Read Only` 再让它建文件就能复现。

> 💡 一句话总结：照着「`/` 看全集 → `/status` 看配置 → `Ctrl+L` / `!` 体会快捷键 → `/diff` 审改动 → `/exit` 退」走一遍 CLI，再分别在 App、IDE 里打个 `/` 对比命令表——亲手确认「三处不一样」「快捷键和斜杠各管一摊」，记得最牢。

## 07 小结

这一篇把斜杠命令和快捷键从「你每天敲、却没摸透的入口」给你掀开看了——**它是 Codex 会话的控制台，从切模型、清上下文到审改动，一个 `/` 全管；再配一把快捷键，操作快上一截**。

把核心要点串成一张表，揣兜里：

| 你想搞清的事 | 答案 | 关键点 |
|------------|------|--------|
| 斜杠命令是什么 | Codex 会话的控制指令 | 管程序本身、不是派活；只在消息开头算数 |
| CLI 命令怎么记 | 按「你在哪一步」分组查 | 搭场子 / 干活中 / 交付前 / 杂项；打 `/` 见真实全集 |
| `/clear` `/new` `/compact` 咋分 | 清屏换上下文 / 不清屏换上下文 / 压缩续聊 | 换任务用前俩，续任务用 `/compact` |
| 桌面 App 能用啥 | 现列 6 条 | `/status` `/review` `/plan` `/goal` `/mcp` `/feedback` |
| IDE 扩展能用啥 | 现列 8 条 | 独有 `/local` `/cloud` `/auto-context` |
| IDE 命令面板那套 | 编辑器层动作 | `chatgpt.addToThread`、`implementTodo` 等，自己绑键 |
| CLI 快捷键记哪些 | `Ctrl+L` `Ctrl+O` `Tab` `Ctrl+G` `!` | `Ctrl+L` ≠ `/clear`（不丢对话） |
| 想改键 | `/keymap`（CLI）、键盘快捷方式（IDE / App） | 新手先用熟默认的 |

**你现在应该能：** 在任意一个入口打 `/` 就看懂那张菜单、按「你在哪一步」找到该用的命令；分得清 CLI、桌面 App、IDE 三处的斜杠命令为啥不一样、各有哪些；记牢 `Ctrl+L` 和 `/clear` 别按混；还知道 IDE 命令面板里那套「指着代码动手」的 Codex 动作怎么绑快捷键。

回头看开头那两周的「重启大法」——**说到底就是没翻过那个 `/` 菜单**。这一篇之后，但凡你想对 Codex 本身做点啥，第一反应该是先打个斜杠看看有没有现成的开关。**这点习惯，能帮你省下那一大把白费的时间。**
